# 前端设计规范

## 克制设计原则

**规则：** 避免与功能无关的提示性描述、装饰性标签、解释性文案。

**具体要求：**
- 界面只展示决策所需信息
- 不为"看起来完整"而增加内容
- 保持页面简洁、留白充足、信息层次清晰

## 色彩规范

**禁止：** 使用 AI 常见的紫色渐变、靛蓝色调（purple-*, indigo-*, violet-*）

**采用：** 低饱和度色彩体系
- 基础：中性灰、深灰、白色
- 情绪区：偏暖色（红色买入信号、绿色卖出信号）
- 主线区：偏冷色
- 只做轻微层次变化

## 信息密度

**规则：** 首屏直出核心信息

**必须可见：**
- 支撑/压力/触发/失效等关键点位
- 不能被抽象总结替代

**减少层级：**
- 能在原卡片内解决的不新增额外容器
- 概览区或折叠层

## 小白友好

**禁止：**
- 展示 UI 使用指南
- 展示 filter 解释文案
- 冗长的刷新状态描述

**状态标签：** 使用最简短词汇（如"实时刷新"而非"自动实时刷新"）

## Vue 3 + TailwindCSS 模式

**风格：** Composition API + `<script setup>`

**状态管理：** Pinia

**HTTP：** Axios

**组件命名：** PascalCase

**参考：** `frontend/src/` 目录结构
