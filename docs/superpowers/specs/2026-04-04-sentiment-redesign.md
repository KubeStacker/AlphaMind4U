## 1. 现状 Review

### 1.1 架构总览

情绪模块由三层组成：

| 层 | 文件 | 职责 |
|---|------|------|
| **数据层** | `etl/tasks/sentiment_history_task.py`, `etl/sync.py` | 历史回补 + 每日定时计算 |
| **策略层** | `strategy/sentiment/analyst.py`, `config.py` | 12因子评分 + 标签映射 |
| **实时层** | `strategy/sentiment/live_monitor.py` | 盘中指数叠加 + 宏观信号(10Y/Pizza) |

数据流：
```
daily_price / market_index / stock_margin / stock_moneyflow
  → SentimentAnalyst._calculate_fingerprint()
  → SentimentAnalyst._calculate_continuous_score() (50基准, 0-100输出)
  → market_sentiment 表

market_sentiment + 实时指数(sina) + CNBC 10Y + Pizza
  → LiveSentimentMonitor.build_live_overlay()
  → Dashboard 展示
```

## 3. 情绪模块的科学化重构

### 3.1 核心问题：当前评分体系的表述不科学

当前12个"权重"实际是**缩放系数**——每个因子先计算原始值，乘以系数后直接加到50分基准上。这不是真正的加权平均，而是**因子叠加模型**。问题：

1. **量纲不统一**：涨跌停差值(个数)和融资变化率(百分比)直接乘以不同系数后相加，物理意义不清晰
2. **无归一化**：各因子未做标准化处理，系数承担了归一化和加权双重角色
3. **不可解释**：75.0的promotion和0.25的limit_diff，用户无法直观理解为什么差300倍

### 3.2 新的评分模型设计

采用**分层因子模型**，将12个因子按A股情绪特征分为4个维度：

```
情绪总分 (0-100)
├── 赚钱效应 (30%)
│   ├── 涨跌停差值        → 市场赚钱窗口
│   ├── 涨停晋级率        → 接力意愿
│   ├── 炸板数            → 封板质量(反向)
│   └── 修复力度          → 自我修复能力
├── 量价结构 (25%)
│   ├── 市场广度          → 普涨/普跌
│   ├── 成交活跃度        → 量能配合
│   ├── 新高/新低比       → 趋势结构
│   └── 连板高度          → 空间标杆
├── 资金态度 (25%)
│   ├── 融资余额变化      → 杠杆情绪
│   ├── 主力净流入占比    → 机构态度
│   └── 指数共振          → 权重股方向
└── 风险预警 (20%)
    └── 波动率z-score     → 恐慌程度(反向)
```

每个维度内部先做**百分位排名**（基于历史数据），得到0-100的标准分，再按维度权重加权。

### 3.3 标签体系优化

| 分数 | 标签 | 仓位建议 | 操作策略 | A股特征 |
|------|------|----------|----------|---------|
| 85+ | 沸腾 | ≤30% | 减仓防守，不追高 | 涨停潮+缩量加速，次日易分化 |
| 70-84 | 高热 | 50-70% | 持有龙头，不新开仓 | 主线清晰，赚钱效应强 |
| 55-69 | 修复 | 50-70% | 试探建仓，跟随主线 | 冰点后首次回暖，确认信号 |
| 42-54 | 拉锯 | 30-50% | 控制仓位，做T为主 | 涨跌互现，无明确方向 |
| 25-41 | 低温 | ≤30% | 防守为主，等待信号 | 跌停增多，接力失败率高 |
| <25 | 冰点 | 0-10% | 空仓等待反转 | 千股跌停/大面积跌停，恐慌极致 |

### 3.4 A股特殊考量

A股情绪有其独特性，必须在模型中体现：

1. **涨跌停制度**：10%/20%涨跌停限制使得情绪极端化，涨停/跌停数量是最直接的情绪指标
2. **T+1交易**：当日买入次日才能卖出，导致情绪有惯性，昨日涨停晋级率极其重要
3. **题材炒作**：A股以题材/概念驱动为主，主线板块的持续性决定情绪质量
4. **政策敏感**：政策底/市场底/情绪底的三段式见底过程，需要结合宏观信号
5. **融资融券**：融资余额变化反映散户杠杆情绪，是重要的反向指标
6. **北向资金**：外资流向影响权重股和指数，间接影响情绪

### 3.5 实时叠加层优化

盘中实时叠加应区分**交易时段**和**非交易时段**：

- **交易时段**：实时指数 + 10Y/Pizza → 动态调整情绪分
- **非交易时段**：返回最近一个交易日的情绪分，不做实时调整
- **海外风险提示**：10Y美债和Pizza指数作为**独立风险信号**展示，不直接加减分

## 4. Dashboard 拆分方案

### 4.1 拆分目标

- 将 `Dashboard.vue` (1841行) 拆为两个独立组件
- 两个组件在同一页面展示
- 保持现有功能不变
- 遵循AGENTS.md的前端设计规范（克制表达、高级感、低饱和度色彩）

### 4.2 拆分结果

```
frontend/src/views/Dashboard.vue          → 父容器，引入两个子组件
frontend/src/components/dashboard/SentimentPanel.vue   → 情绪驾驶舱
frontend/src/components/dashboard/MainlinePanel.vue    → 主线作战板
```

### 4.3 组件职责

**SentimentPanel.vue** (~500行):
- 情绪仪表盘（圆形仪表 + 分数 + 标签）
- 5个指标卡（市场广度、涨停/跌停、上证实时、10Y美债、Pizza指数）
- 风险预测面板
- 历史弹窗（ECharts折线图）
- 自动刷新逻辑
- 手动同步按钮

**MainlinePanel.vue** (~400行):
- 主线列表（最多3条）
- 龙头评分卡片
- 方向卡片（点击展开Top5龙头）
- 轮动正文

**Dashboard.vue** (~100行):
- 页面布局容器
- 引入 SentimentPanel 和 MainlinePanel
- 响应式布局（上下排列）

### 4.4 布局设计

```
┌─────────────────────────────────────────┐
│  Dashboard                               │
├─────────────────┬───────────────────────┤
│                 │                        │
│  SentimentPanel │   MainlinePanel       │
│  (情绪驾驶舱)    │   (主线作战板)         │
│                 │                        │
│  [仪表盘]       │   [主线1] ████████     │
│  [指标卡×5]     │   [主线2] ██████       │
│  [风险预测]     │   [主线3] ████         │
│                 │                        │
└─────────────────┴───────────────────────┘
```

左右分栏布局，情绪区在左（约40%宽度），主线区在右（约60%宽度）。移动端自动切换为上下排列。

### 4.5 状态管理

暂不引入Pinia store。两个组件各自管理内部状态，通过props从父组件接收共享数据（如loading状态）。

### 4.6 样式规范

- 情绪区：暖色调基底（低饱和红色/橙色），买入信号红色，卖出信号绿色
- 主线区：冷色调基底（低饱和蓝色/青色）
- 不使用紫色/靛蓝色渐变
- 统一圆角(8px)、阴影、间距(16px)
- 字体：标题14px，正文12px，数值16px
