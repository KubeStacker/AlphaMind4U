# Jarvis 策略系统详述 (Detailed Strategy System Documentation)

本文档旨在为 Jarvis 系统的策略体系提供完整的技术说明。

---

## 1. 体系架构概述

Jarvis 策略引擎采用三层递进架构，旨在实现从宏观择时到微观选股的闭环。

### 层级一：市场情绪分析 (`/backend/strategy/sentiment`)
*   **目标**：解决“何时买”的问题。
*   **核心逻辑**：基于全市场 5000+ 只股票的涨跌分布、连板高度、跌停家数等指标，构建自适应情绪导数模型。
*   **关键产出**：每日生成情绪分值与交易预案（BUY/SELL/HOLD/WATCH）。

### 层级二：主线板块分析 (`/backend/strategy/mainline`)
*   **目标**：解决“买什么方向”的问题。
*   **核心逻辑**：通过概念共振算法，计算各板块的赚钱效应评分。解决“一票多概念”的归类偏向问题。
*   **关键产出**：识别当日及近期的核心领涨主线（如：AI、低空经济、高股息等）。

### 层级三：个股推荐引擎 (`/backend/strategy/recommend`)
*   **目标**：解决“具体买哪只”的问题。
*   **核心逻辑**：插件化架构，支持多策略并列。

## 2. 核心策略详解

### 2.1 Alpha 动量增强策略 (Alpha Momentum Strategy)

该策略是系统的核心选股逻辑，聚焦于**处于强势上升通道且具备业绩/动量支撑**的标的。

#### 核心因子：
1.  **RPS (相对价格强度)**：
    *   **RPS_250**：股价一年内的市场排名，反映长线趋势。
    *   **RPS_20**：近一个月的市场排名，捕捉短线爆发力。
    *   *判定标准*：RPS_250 > 90 或 RPS_20 > 85。
2.  **Sharpe Ratio (夏普比率)**：
    *   通过计算近 60 日收益波动比，优先选择上涨路径平滑、回撤可控的“走势龙”。
3.  **Bias (乖离率)**：
    *   限制 20 日均线乖离率，防止追高，寻找“回踩支撑”的介入点。

#### 智能优化机制：
*   **数据韧性**：自动寻找最新的有效因子交易日。
*   **动态回退**：若主线内无合适标的，会自动放宽条件或执行全市场扫描，确保策略不空转。

### 2.2 回调反转策略 (Mean Reversion Strategy)

专门捕捉“牛回头”或“二波起涨”的机会。

#### 核心逻辑：
1.  **历史强度校验**：标的必须在过去一年内有过 RPS > 90 的高光时刻（证明过股性）。
2.  **深度回撤**：股价较年度高点回撤 30% - 50% 左右。
3.  **均线支撑**：股价回踩至 MA60 (季线) 或 MA120 (半年线) 附近。

## 3. 开发者指南

### 如何新增一个选股插件？
1.  在 `/backend/strategy/recommend/plugins/` 下新建 Python 文件。
2.  继承 `BaseStrategyPlugin` 类。
3.  实现 `run()` 和 `recommend()` 方法。
4.  在 `documentation` 属性中定义该策略的逻辑说明（此说明会自动同步至系统前端展示）。
